В данной папке лежат МУ, в которых реализуются любые функции управления.  Модули подразделяются на два типа:
IFM_ВашеИмя - модуль управления интерфейсом (InterFace Management);
OWM_ВашеИмя - модуль управления внешней средой, или окружающим миром (Out World Management), то есть не самой ИСУ.

Отличие модуля управления интерфейсом от модулей управления внешней средой лишь в том, что запускается только один из них, который управляет используемым интерфейсом.

В настройках ИСУ следует указывать имя модуля управления интерфейсом. Если такого модуля нет для испоьлзуемого интерфейса, то просто не указываем этот параметр.

В конец необходимо вставить следующую функцию:
  def ListFuncIMS(ModuleSettings):
    argument_Esperanto = {
      0: {'dolar': 'USD', 'rubl': 'RUB', 'eŭro': 'EUR'},
      1: {'rusi': 'Russia', 'belorusi': 'Belarus'}
      }
    ListFunc = [
      {
       'function': (GetCourse, 2),
       'Esperanto': {'verb': u'montr', 'noun': u'kurs'}, # если не указать существительное, то функция прикрепится к глаголу. То есть вызов функции будет так: montrus, а не так: montrus kurson
       'argument_Esperanto':argument_Esperanto,
       'checking': {0: True, 1: True}
      }
    ]
    return ListFunc

"argument_ИмяЕЯ" - значение - словарь. В качестве клюей - номера каждого аргумента. Значения - словарь, в котором содержатся ассоциации вида ЕЯ : Строка. При конвертации из ЕЯ в Вя, проверяется наличие слова-аргумента в этом словаре.

При отсутствии слова-аргумента:
  Если номер аргумента в ключе 'cheking' равен True, то в результате выдаётся ошибка ключа.
  Если равен False, то слово-аргумент передаётся напрямуюю в функцию, минуя этот словарь.

В будущем можно реализовать обучение: При отсутствии слова-аргумента ИСУ может попросить придумать для него имя, чтобы занести эту пару в словарь для дальнейшего удобства :)
